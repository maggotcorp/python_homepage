Dumb Phone

A comprehensive network security application for Android and iOS that provides real-time protection against threats, monitoring network traffic, and blocking malicious connections through two distinct firewall modes.

Features

  • Dumb Phone Mode: High-security mode that blocks all non-essential traffic by default, allowing only voice and text-related communications.
  • Soft Firewall Mode: Intelligent monitoring and blocking based on customizable rules and threat intelligence.
  • Country Blacklisting: Block all traffic to/from specific countries known for high cyber-threat activity.
  • Real-time Logging: Monitor all allowed and blocked connections with detailed packet information.
  • Process Monitoring: View running processes and their associated network activity.
  • Custom Rules: Create your own IP, port, or protocol-based firewall rules.
  • Threat Intelligence: Pre-loaded database of high-risk countries and IP ranges.
  • Log Retention: Configurable log retention with automatic cleanup.

Getting Started

Prerequisites

  • Flutter SDK (>= 3.16.0)
  • Android SDK (with NDK support)
  • Xcode (for iOS builds)
  • ADB (for Android device management)

Installation

  1. Clone the repository: 
    git clone 
cd dumb_phone
  2. Install dependencies: 
    flutter pub get
  3. Build the application:

Android: 

make android

iOS: 

make ios

  1. Run on a connected device:

Android: 

make run-android

Development

The project uses a standard Flutter structure with a clean architecture approach: - lib/data: Data models, database helpers, and services. - lib/ui: Screens and UI components. - lib/viewmodel: Application state and logic. - android/: Native Android implementation for VPN services. - ios/: Native iOS implementation.

Common Commands

  • make lint: Run Flutter analyzer.
  • make format: Format the code.
  • make test: Run unit tests.
  • make clean: Clean build artifacts.
  • make permissions: Grant necessary permissions via ADB (for development).

Google Play Store Deployment

Signing Configuration

The project is configured for release signing. To set up signing:

  1. Create a keystore file: 
    keytool -genkey -v -keystore dumb-phone-release.keystore -alias dumb-phone -keyalg RSA -keysize 2048 -validity 10000
  2. Create a keystore.properties file in the project root: 
    storeFile=dumb-phone-release.keystore
storePassword=your_store_password
keyAlias=dumb-phone
keyPassword=your_key_password
  3. Build a signed release: 
    flutter build apk --release

ProGuard

The project includes ProGuard rules for code obfuscation and optimization. The release build automatically applies these rules.

iOS Deployment

Code Signing

  1. Open the iOS project in Xcode: 
    open ios/Runner.xcworkspace
  2. Configure signing in Xcode: - Select the Runner target - Go to Signing & Capabilities - Select your team and provisioning profile
  3. Build for release: 
    flutter build ios --release

Architecture

The application follows the MVVM (Model-View-ViewModel) pattern:

  • Models: Data structures for firewall rules, logs, and country blacklists.
  • ViewModels: Business logic and state management using Flutter's ValueNotifier.
  • Views: UI screens and widgets.
  • Services: Native Android/iOS services for VPN and firewall functionality.

Database Schema

The application uses SQLite for local storage with the following tables:

  • blocklist_entries: Country blacklist configurations.
  • country_ip_ranges: IP ranges for blacklisted countries.
  • firewall_rules: Custom firewall rules.
  • firewall_logs: Network activity logs.

Privacy

All processing and logging happen locally on your device. Your network traffic data is never sent to external servers.
